Room Booking Meeting Scheduler - 2015

This project was really an introduction to network based applications and multi-threading. I had lots of fun making and designing it. Although, everytime I look back at this project, it reminds me of how little I knew about programming and its technologies back then.

Overall Technology:

Java – Swing (Java) – Notepad Database (I still find this hilarious)

> What?

Remotely connect to a central server and book meeting with other users. To implement this, two programs had to be created: the server and the client GUI.

RBMS Server

The server was definately the hardest part of this project. It was the first "big" program I created from scracth.

It's responsibility included managing all the connections with the client applications, storing all the data in a central location and managing all the business rules.

Client GUI

The client GUI was programmed in Java using Swing. The GUIs primary use was to make it easier to use as well as to be able to add validation rules on the user inputs.
